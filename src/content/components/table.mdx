## Overview

The Table system provides a comprehensive set of components for creating rich, interactive data tables. It includes semantic HTML table elements with built-in styling, sorting capabilities, pagination, search functionality, and Alpine.js integration for dynamic interactions.

### Complete Table Architecture

The table system consists of these core components:

- **Table**: Main container with variants and responsive behavior
- **TableHeader**: Header section with column definitions  
- **TableBody**: Data rows container with interactive states
- **TableRow**: Individual rows with selection and hover effects
- **TableHead**: Column headers with sorting capabilities
- **TableCell**: Data cells with flexible alignment and content
- **TablePagination**: Advanced pagination with page navigation and size selection
- **TableToolbar**: Comprehensive toolbar with search, filters, and actions

## Component Architecture

### Table (Main Container)

The root Table component provides the foundation with variants and responsive behavior.

**Props:**
- `variant`: `'default' | 'bordered' | 'striped' | 'hoverable'` - Visual style
- `density`: `'compact' | 'normal' | 'comfortable'` - Row spacing
- `size`: `'sm' | 'md' | 'lg'` - Text size
- `responsive`: `boolean` - Enable horizontal scrolling on small screens

### TableHeader

Wraps the table header section (thead) with consistent styling and muted background.

**Props:**
- `class`: `string` - Additional CSS classes

**Default Styling:**
- `border-b border-border` - Bottom border separator
- `bg-muted/50` - Subtle background color
- Proper semantic HTML `<thead>` element

### TableBody

Contains data rows with dividers and interactive states.

**Props:**
- `class`: `string` - Additional CSS classes

**Default Styling:**
- `divide-y divide-border` - Row dividers
- Proper semantic HTML `<tbody>` element
- Inherits table styling from parent Table component

### TableRow

Individual table rows with selection and hover states.

**Props:**
- `selected`: `boolean` - Whether the row is selected
- `class`: `string` - Additional CSS classes

**Default Styling:**
- `border-b transition-colors` - Bottom border and smooth transitions
- `hover:bg-muted/50` - Hover effect
- `data-[state=selected]:bg-muted` - Selected state styling
- When `selected={true}`: adds `bg-muted/50` class

### TableHead

Column headers with sorting capabilities and alignment.

**Props:**
- `sortable`: `boolean` - Enable sorting indicator
- `align`: `'left' | 'center' | 'right'` - Text alignment
- `width`: `string` - Column width (Tailwind width class number)
- `class`: `string` - Additional CSS classes

**HTML Attributes:**
- `colspan`: `number` - Number of columns to span
- `rowspan`: `number` - Number of rows to span
- `scope`: `string` - Accessibility scope attribute

**Default Styling:**
- `h-12 px-4 text-left align-middle font-medium text-muted-foreground` - Basic styling
- `[&:has([role=checkbox])]:pr-0` - Reduced padding for checkboxes
- Text alignment based on `align` prop
- When `sortable={true}`: adds cursor-pointer and hover effects
- When `width` prop: adds responsive width class

### TableCell

Data cells with flexible alignment and content support.

**Props:**
- `align`: `'left' | 'center' | 'right'` - Text alignment  
- `class`: `string` - Additional CSS classes

**HTML Attributes:**
- `colspan`: `number` - Number of columns to span
- `rowspan`: `number` - Number of rows to span

**Default Styling:**
- `p-4 align-middle` - Padding and vertical alignment
- `[&:has([role=checkbox])]:pr-0` - Reduced padding for checkboxes
- `text-sm` - Small text size
- Text alignment based on `align` prop

### TablePagination

Advanced pagination with page navigation and size selection.

**Props:**
- `showInfo`: `boolean` - Show pagination information
- `showPageSize`: `boolean` - Show page size selector
- `pageSizeOptions`: `number[]` - Available page sizes
- `class`: `string` - Additional CSS classes

**Required Alpine.js Properties:**
- `currentPage`: Current active page number
- `totalPages`: Total number of pages
- `pageSize`: Current page size
- `filteredData`: Object with `length` property for item count
- `goToPage(page)`: Navigate to specific page
- `nextPage()`: Go to next page
- `previousPage()`: Go to previous page
- `setPageSize(size)`: Change page size

### TableToolbar

Comprehensive toolbar with search, filters, and actions.

**Props:**
- `searchPlaceholder`: `string` - Search input placeholder
- `showSearch`: `boolean` - Show search input
- `showActions`: `boolean` - Show actions section
- `class`: `string` - Additional CSS classes

**Slots:**
- `filters`: Custom filter controls
- `actions`: Regular action buttons
- `selection-actions`: Actions shown when rows are selected

**Required Alpine.js Properties:**
- `searchQuery`: Current search query
- `filter()`: Function to filter data
- `selectedCount`: Number of selected rows
- `hasSelection`: Whether any rows are selected
- `clearSelection()`: Function to clear all selections

## Best Practices

### Performance
- Use pagination for large datasets (>100 items)
- Implement debounced search to avoid excessive filtering
- Consider virtual scrolling for extremely large datasets
- Use proper `x-key` attributes in Alpine.js templates

### User Experience
- Provide clear visual feedback for all interactions
- Include loading states for async operations
- Show helpful empty states with actionable messages
- Use consistent iconography and terminology
- Maintain selection state across pagination

### Data Management
- Validate and sanitize user input
- Handle edge cases (null values, mixed data types)
- Provide clear error messages
- Consider implementing optimistic updates for better perceived performance

The table system provides a complete, accessible, and performant foundation for building rich data tables in your Astro applications.
