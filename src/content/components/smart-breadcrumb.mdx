## Overview

The SmartBreadcrumb component automatically generates breadcrumb navigation with intelligent overflow handling. When there are too many items, it shows ellipsis with a dropdown containing the hidden items.

## URL-Based Auto Generation

You can create breadcrumbs automatically from the current URL path:

<Tabs defaultValue="code" class="h-auto">
    <TabsList variant="pills" slot="list">
        <TabsTrigger value="code" variant="pills">
            Code
        </TabsTrigger>
    </TabsList>
    <TabsContent value="code">
        <Code
            language="astro"
            code={`---
import SmartBreadcrumb from '@/components/ui/navigation/breadcrumbs/SmartBreadcrumb.astro';

// Auto-generate breadcrumbs from URL
const url = new URL(Astro.request.url);
const pathSegments = url.pathname.split('/').filter(Boolean);

const breadcrumbItems = [
    { label: 'Home', href: '/' },
    ...pathSegments.map((segment, index) => {
        const href = '/' + pathSegments.slice(0, index + 1).join('/');
        const label = segment.charAt(0).toUpperCase() + segment.slice(1).replace(/-/g, ' ');
        return { label, href };
    })
];
---
<SmartBreadcrumb 
    x-data={\`{ items: \${JSON.stringify(breadcrumbItems)} }\`}
/>`}
        />
    </TabsContent>
</Tabs>

## API Reference

### Props

| Prop         | Type                                           | Default      | Description                               |
| ------------ | ---------------------------------------------- | ------------ | ----------------------------------------- |
| `variant`    | `'default' | 'minimal' | 'compact'`          | `'default'`  | Visual style variant                      |
| `size`       | `'sm' | 'md' | 'lg'`                        | `'md'`       | Size of the breadcrumb                    |
| `overflow`   | `'fixed' | 'wrap' | 'scroll'`               | `'fixed'`    | Overflow handling strategy                |
| `aria-label` | `string`                                       | `'breadcrumb'` | ARIA label for accessibility           |
| `maxItems`   | `number`                                       | `5`          | Maximum items before showing ellipsis    |
| `separator`  | `'chevron' | 'slash' | 'dot' | 'arrow'`      | `'chevron'`  | Separator icon between items             |
| `class`      | `string`                                       | -            | Additional CSS classes                    |
| `...`        | `any`                                          | -            | Alpine.js props passed through            |

### Alpine.js Data Structure

The component expects an `items` array in Alpine.js data:

```typescript
{
  items: Array<{
    label: string;    // Display text for the breadcrumb
    href: string;     // URL for the breadcrumb link
  }>
}
```

### Internal Alpine.js Properties

| Property              | Type      | Description                                  |
| --------------------- | --------- | -------------------------------------------- |
| `showDropdown`        | `boolean` | Controls ellipsis dropdown visibility        |
| `processed.displayItems` | `array` | Items to display (includes ellipsis)        |
| `processed.hidden`    | `array`   | Items hidden in the ellipsis dropdown       |
| `processed.shouldShowEllipsis` | `boolean` | Whether to show ellipsis             |

## Features

- **Intelligent Overflow**: Automatically handles too many breadcrumb items
- **Dropdown Navigation**: Hidden items accessible via ellipsis dropdown
- **Multiple Variants**: Different visual styles for various contexts
- **Responsive Design**: Adapts to different screen sizes
- **Accessibility**: Proper ARIA labels and semantic HTML
- **Flexible Data**: Works with Alpine.js data or slot content
- **Auto-generation**: Can be generated from URL paths
- **Custom Separators**: Multiple separator icon options

## Usage Notes

- The component automatically shows first, second-to-last, and last items when overflow occurs
- Hidden items in the middle are accessible via the ellipsis dropdown
- When no Alpine.js items data is provided, it falls back to slot content
- The dropdown closes when clicking outside or selecting an item
- Perfect for deep navigation hierarchies in documentation sites
- Integrates seamlessly with Astro's file-based routing
