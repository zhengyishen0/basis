## Overview

Breadcrumbs provide navigation hierarchy showing the user's location within a website or application. The component system includes a basic `Breadcrumb` container, individual `BreadcrumbItem` components, dynamic `BreadcrumbItems` for Alpine.js x-for loops, and `SmartBreadcrumb` for automatic ellipsis handling with dropdown overflow.

The components support multiple separator types (chevron, slash, dot, arrow), custom icons, and responsive design patterns. All components integrate seamlessly with Alpine.js for dynamic path generation and state management.

## API Reference

### Breadcrumb

The main container component for breadcrumb navigation.

#### Props

| Prop         | Type                              | Default        | Description                            |
| ------------ | --------------------------------- | -------------- | -------------------------------------- |
| `variant`    | `'default', 'minimal', 'compact'` | `'default'`    | Visual variant of the breadcrumb       |
| `size`       | `'sm', 'md', 'lg'`                | `'md'`         | Size of the breadcrumb container       |
| `overflow`   | `'fixed', 'scroll', 'hidden'`     | `'fixed'`      | Overflow handling behavior             |
| `aria-label` | `string`                          | `'breadcrumb'` | Accessibility label for the navigation |
| `class`      | `string`                          | -              | Additional CSS classes                 |

### BreadcrumbItem

Individual breadcrumb item component.

#### Props

| Prop        | Type                                 | Default     | Description                                                      |
| ----------- | ------------------------------------ | ----------- | ---------------------------------------------------------------- |
| `href`      | `string`                             | -           | URL for the breadcrumb link. If omitted, renders as current page |
| `separator` | `'chevron', 'slash', 'dot', 'arrow'` | `'chevron'` | Type of separator to show after this item                        |
| `icon`      | `string`                             | -           | Icon identifier to display alongside text                        |
| `item`      | `object`                             | -           | Dynamic item data for Alpine.js integration                      |
| `class`     | `string`                             | -           | Additional CSS classes                                           |

#### Slots

| Slot        | Description                                         |
| ----------- | --------------------------------------------------- |
| default     | Content of the breadcrumb item                      |
| `separator` | Custom separator element (overrides separator prop) |

### BreadcrumbItems

Template component for rendering breadcrumb items dynamically with Alpine.js x-for.

#### Props

| Prop        | Type                                 | Default     | Description                  |
| ----------- | ------------------------------------ | ----------- | ---------------------------- |
| `separator` | `'chevron', 'slash', 'dot', 'arrow'` | `'chevron'` | Separator type for all items |
| `icon`      | `string`                             | -           | Icon for all items           |
| `class`     | `string`                             | -           | Additional CSS classes       |
| `x-for`     | `string`                             | -           | Alpine.js x-for expression   |
| `x-key`     | `string`                             | `'item.id'` | Alpine.js x-key expression   |

#### Slots

| Slot    | Description                               |
| ------- | ----------------------------------------- |
| default | Content template for each breadcrumb item |

### SmartBreadcrumb

Advanced breadcrumb component with automatic ellipsis handling for long paths.

#### Props

| Prop         | Type                                 | Default        | Description                           |
| ------------ | ------------------------------------ | -------------- | ------------------------------------- |
| `variant`    | `'default', 'minimal', 'compact'`    | `'default'`    | Visual variant                        |
| `size`       | `'sm', 'md', 'lg'`                   | `'md'`         | Size of the breadcrumb                |
| `maxItems`   | `number`                             | `5`            | Maximum items to show before ellipsis |
| `separator`  | `'chevron', 'slash', 'dot', 'arrow'` | `'chevron'`    | Separator type                        |
| `aria-label` | `string`                             | `'breadcrumb'` | Accessibility label                   |
| `class`      | `string`                             | -              | Additional CSS classes                |

## Alpine.js Integration

All breadcrumb components integrate seamlessly with Alpine.js for dynamic behavior:

### Data Structure

```javascript
// Expected data structure for dynamic breadcrumbs
{
  items: [
    {
      id: string,           // Unique identifier
      label: string,        // Display text
      href?: string,        // Link URL (optional for current page)
      current?: boolean,    // Whether this is the current page
      icon?: string         // Icon identifier (optional)
    }
  ]
}
```

### SmartBreadcrumb Features

- **Automatic ellipsis**: Shows "..." when path exceeds `maxItems`
- **Dropdown overflow**: Click ellipsis to see hidden items
- **Responsive patterns**: Adapts to different screen sizes
- **Keyboard navigation**: Full keyboard accessibility

### Events

The breadcrumb components don't emit custom events but integrate with standard navigation patterns and Alpine.js reactivity.

## Accessibility

- Uses semantic `<nav>` and `<ol>` elements
- Proper `aria-label` and `aria-current` attributes
- Keyboard navigation support in SmartBreadcrumb dropdown
- Screen reader friendly with proper markup structure
- Focus management for interactive elements

## Best Practices

1. **Use semantic markup**: Always wrap breadcrumbs in the provided nav element
2. **Current page indication**: Use `current: true` or omit `href` for current page
3. **Reasonable depth**: Keep breadcrumb paths manageable (5-7 levels max)
4. **Consistent separators**: Use the same separator throughout your application
5. **Mobile considerations**: Consider using SmartBreadcrumb for responsive designs
6. **Icon usage**: Use icons sparingly and consistently (typically just for home)
