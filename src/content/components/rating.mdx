## Alpine.js Integration

The Rating component integrates seamlessly with Alpine.js for dynamic behavior:

<Tabs defaultValue="preview" class="h-auto">
    <TabsList variant="pills" slot="list">
        <TabsTrigger value="preview" variant="pills">
            Preview
        </TabsTrigger>
        <TabsTrigger value="code" variant="pills">
            Code
        </TabsTrigger>
    </TabsList>
    <TabsContent value="preview" preview class="flex flex-col gap-4 items-center">
        <div x-data="{ rating: 0 }" class="flex flex-col gap-4 items-center p-4 border border-border rounded-lg">
            <Rating 
                x-model="rating" 
                @rating-changed="console.log('Rating changed to:', $event.detail)"
                initialValue={0}
            >
                <span class="text-sm text-muted-foreground">Dynamic rating</span>
            </Rating>
            <p class="text-sm" x-text="`Current rating: ${rating}`"></p>
        </div>
    </TabsContent>
    <TabsContent value="code">
        <Code
            language="astro"
            code={`---
import Rating from '@/components/ui/forms/Rating.astro';
---
<div x-data="{ rating: 0 }" class="flex flex-col gap-4 items-center">
    <Rating 
        x-model="rating" 
        @rating-changed="console.log('Rating changed to:', $event.detail)"
        initialValue={0}
    >
        <span class="text-sm text-muted-foreground">Dynamic rating</span>
    </Rating>
    <p class="text-sm" x-text="\`Current rating: \${rating}\`"></p>
</div>`}
        />
    </TabsContent>
</Tabs>

## API Reference

### Props

| Prop           | Type      | Default | Description                                      |
| -------------- | --------- | ------- | ------------------------------------------------ |
| `maxStars`     | `number`  | `5`     | Maximum number of stars                          |
| `initialValue` | `number`  | `0`     | Initial rating value                             |
| `disabled`     | `boolean` | `false` | Disable user interaction                         |
| `readonly`     | `boolean` | `false` | Make the rating read-only                        |
| `name`         | `string`  | -       | Form field name for hidden input                |
| `class`        | `string`  | -       | Additional CSS classes                           |
| `...`          | `any`     | -       | Alpine.js props passed through to the container |

### Events

| Event            | Detail   | Description                        |
| ---------------- | -------- | ---------------------------------- |
| `rating-changed` | `number` | Fired when the rating value changes|

### Alpine.js Data

The component exposes the following Alpine.js data:

| Property   | Type      | Description                           |
| ---------- | --------- | ------------------------------------- |
| `value`    | `number`  | Current rating value                  |
| `stars`    | `number`  | Currently displayed stars (for hover) |
| `disabled` | `boolean` | Whether the component is disabled     |
| `readonly` | `boolean` | Whether the component is readonly     |
| `maxStars` | `number`  | Maximum number of stars               |

### Alpine.js Methods

| Method           | Parameters | Description                    |
| ---------------- | ---------- | ------------------------------ |
| `rate(star)`     | `number`   | Set the rating to a specific value |
| `reset()`        | -          | Reset the rating to 0          |
| `hoverStar(star)`| `number`   | Handle star hover              |
| `mouseLeftStar()`| -          | Handle mouse leave             |

## Features

- **Interactive Rating**: Click stars to set rating values
- **Hover Effects**: Visual feedback on star hover
- **Readonly Mode**: Display ratings without user interaction
- **Form Integration**: Hidden input for form submission
- **Custom Star Count**: Support for different rating scales
- **Alpine.js Integration**: Full Alpine.js compatibility with x-model
- **Accessibility**: Proper ARIA labels and keyboard support
- **Custom Events**: Emits rating-changed events for integration
- **Reset Functionality**: Click slot content to reset rating

## Usage Notes

- The component uses Lucide star icons that fill on selection
- Hover effects provide visual feedback before selection
- The slot content can be clicked to reset the rating
- Form integration creates a hidden input with the specified name
- Alpine.js x-model directive allows two-way data binding
- Custom events bubble up for integration with parent components
