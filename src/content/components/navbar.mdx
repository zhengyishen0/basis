## Overview

## Mobile-Responsive Layout

The Navbar automatically hides navigation links on mobile devices (hidden on screens smaller than `md` breakpoint) and shows them on desktop. You can implement a mobile menu toggle using Alpine.js:

## API Reference

### Props

| Prop     | Type      | Default | Description                                    |
| -------- | --------- | ------- | ---------------------------------------------- |
| `sticky` | `boolean` | `true`  | Whether the navbar should stick to the top    |
| `class`  | `string`  | -       | Additional CSS classes for the nav element    |
| `...`    | `any`     | -       | Alpine.js props passed through to the header  |

### Slots

| Slot      | Description                                         |
| --------- | --------------------------------------------------- |
| `brand`   | Logo, brand name, or brand-related content         |
| `links`   | Navigation links (hidden on mobile by default)     |
| `actions` | Action buttons, user menu, or other action items   |

## Features

- **Responsive Design**: Automatically adapts to mobile and desktop layouts
- **Sticky Positioning**: Optional sticky header with backdrop blur effects
- **Slot-Based Architecture**: Flexible content organization with named slots
- **Alpine.js Integration**: Full support for Alpine.js directives and data
- **Accessible**: Semantic HTML structure with proper navigation landmarks
- **Customizable**: Easy styling with CSS classes and Tailwind utilities

## Usage Notes

- Navigation links are automatically hidden on mobile devices (screens smaller than `md` breakpoint)
- The component uses backdrop blur effects for modern browser support
- Brand slot typically contains logos and company names
- Actions slot is perfect for login buttons, user avatars, or mobile menu toggles
- Alpine.js integration allows for dynamic behavior like mobile menu toggles
