## Overview

The List component creates a vertical container for organizing items with gap spacing, while ListItems provides row-oriented layout for individual list elements. Together they create flexible, scrollable lists perfect for navigation menus, item lists, and data displays.

**List** is the container component that:
- Creates vertical lists with gap spacing between items
- Has default `max-h-screen` constraint with `auto` overflow for scrolling
- Handles different overflow behaviors (auto, fixed, expand)

**ListItems** is the item component that:
- Provides horizontal row layout for list content
- Minimum height of 2.5rem with center alignment
- Designed for Alpine.js `x-for` integration
- Uses `justify-between` for natural item distribution

## API Reference

### List Component

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `overflow` | `'auto' | 'fixed' | 'expand' | 'wrap' | 'show'` | `'auto'` | Controls overflow behavior |
| `class` | `string` | - | Additional CSS classes |
| `...alpineProps` | `any` | - | Alpine.js attributes (x-data, x-for, etc.) |

#### Overflow Behaviors

- **`auto`** (default): Vertical scrolling when content exceeds container height
- **`fixed`**: Content is clipped at container boundaries
- **`expand`**: Container grows to fit all content (removes max-h-screen)
- **`show`**: Content can spill outside container bounds

#### Default Constraints

- **`max-h-screen`**: Default maximum height constraint (can be overridden with CSS classes)
- **`flex flex-col`**: Vertical layout with flexbox

#### Slots

| Slot | Description |
|------|-------------|
| default | List items to be arranged vertically |

### ListItems Component

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `overflow` | `'auto' | 'fixed' | 'expand' | 'wrap' | 'show'` | `'expand'` | Controls overflow behavior |
| `class` | `string` | - | Additional CSS classes |
| `x-for` | `string` | - | Alpine.js x-for directive |
| `x-key` | `string` | - | Alpine.js x-key directive |
| `...alpineProps` | `any` | - | Other Alpine.js attributes |

#### Base Classes

- **`flex flex-row w-full min-h-[2.5rem] items-center justify-between gap-2`**
- Horizontal layout with minimum height
- Content aligned to center vertically
- Space between first and last elements
- Small gap between adjacent items

#### Overflow Behaviors

- **`expand`** (default): Content determines container size
- **`auto`**: Horizontal scrolling when content overflows
- **`fixed`**: Content is clipped at container boundaries
- **`wrap`**: Flex items wrap to new lines
- **`show`**: Content can spill outside container bounds

#### Slots

| Slot | Description |
|------|-------------|
| default | Content arranged horizontally in the row |

#### Alpine.js Integration

ListItems is designed to work seamlessly with Alpine.js `x-for`:
- Automatically extracts `x-for` and `x-key` from props
- Wraps content in `<template>` tag for Alpine.js iteration
- Passes through all other Alpine.js attributes

## Design Principles

### List Component

List is a **content-driven component** that:
- Only has gap property for spacing between items
- Size determined by internal elements with sensible default constraints
- Handles dynamic/data-driven content
- Default `max-h-screen` with `auto` overflow for vertical scrolling
- Perfect for navigation menus, item lists, and scrollable content
- Can be easily overridden with custom CSS classes for specific height constraints

### ListItems Component

ListItems is a **simple layout container** that:
- Provides row-oriented layout with `flex flex-row`
- Has minimum height of 2.5rem for consistent list item sizing
- Uses `items-center justify-between` for natural item arrangement
- Minimal styling - add custom styles via `class` prop
- Perfect for Alpine.js dynamic content with x-for directives
- For rich styled content containers, use Card component instead

## Working Together

List and ListItems are designed to work together:

1. **List** provides the vertical container with gap spacing and scroll management
2. **ListItems** provides consistent row layout for individual items
3. Both support Alpine.js for dynamic content
4. Both use the content-driven principle where gap and overflow control behavior
5. Combine them for professional-looking lists with minimal markup

Use List alone for simple content, or combine with ListItems for structured, interactive lists with consistent spacing and alignment.
