## Overview

Error is a versatile component for displaying error messages, warnings, informational alerts, and success messages. It provides consistent styling with color-coded variants and flexible sizing options for different contexts.

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'error' | 'warning' | 'info' | 'success'` | `'error'` | Visual variant/theme of the alert |
| `size` | `'sm' | 'md' | 'lg'` | `'md'` | Size of the component |
| `class` | `string` | `undefined` | Additional CSS classes |

### Slots

| Slot | Description |
|------|-------------|
| default | Error message content |

### Variant Colors

| Variant | Background | Border | Text |
|---------|------------|--------|------|
| `error` | `bg-red-50` | `border-red-200` | `text-red-800` |
| `warning` | `bg-yellow-50` | `border-yellow-200` | `text-yellow-800` |
| `info` | `bg-blue-50` | `border-blue-200` | `text-blue-800` |
| `success` | `bg-green-50` | `border-green-200` | `text-green-800` |

### Size Specifications

| Size | Padding | Text Size |
|------|---------|-----------|
| `sm` | `p-3` | `text-sm` |
| `md` | `p-4` | `text-base` |
| `lg` | `p-6` | `text-lg` |

### Alpine.js Pass-through

The Error component supports all Alpine.js directives:

```astro
<Error 
  variant="error"
  x-show="hasError"
  x-transition
  x-on:click="dismissError()"
  x-text="errorMessage"
>
</Error>
```

### Usage Examples

```astro
{/* Basic error */}
<Error>Something went wrong!</Error>

{/* Different variants */}
<Error variant="warning">Warning message</Error>
<Error variant="info">Information message</Error>
<Error variant="success">Success message</Error>

{/* Different sizes */}
<Error size="sm" variant="error">Small error</Error>
<Error size="lg" variant="info">Large info message</Error>

{/* With Alpine.js */}
<Error 
  x-show="showError"
  x-transition
  variant="error"
  x-on:click="showError = false"
>
  Click to dismiss
</Error>

{/* Form validation */}
<input x-model="email" />
<Error 
  x-show="!isValidEmail(email)"
  variant="error"
  size="sm"
>
  Please enter a valid email
</Error>
```

### Features

- **Semantic Variants**: Color-coded variants for different message types
- **Flexible Sizing**: Three size options for different contexts
- **Alpine.js Ready**: Full support for reactive data and event handling
- **Accessible**: Proper color contrast and semantic HTML
- **Customizable**: Easy to extend with additional CSS classes
- **Consistent**: Follows design system spacing and color tokens

### Best Practices

1. **Choose appropriate variants**: Use error for failures, warning for cautions, info for notifications, success for confirmations
2. **Use correct sizing**: Small for inline feedback, medium for standard alerts, large for page-level messages
3. **Provide clear messages**: Write actionable, user-friendly error messages
4. **Handle dismissal**: Consider adding close functionality for non-critical messages
5. **Combine with forms**: Use small size for field-level validation feedback

### Accessibility

- Uses semantic HTML with appropriate ARIA roles
- Color-coded variants meet WCAG contrast requirements  
- Supports keyboard navigation and screen readers
- Text content is descriptive and actionable
