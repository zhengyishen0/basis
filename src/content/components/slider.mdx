## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'primary', 'secondary', 'black', 'white'` | `'primary'` | Color variant for the slider |
| `className` | `string` | - | Additional CSS classes |
| `min` | `number` | `0` | Minimum value |
| `max` | `number` | `100` | Maximum value |
| `value` | `number` | `0` | Current value |
| `buffered` | `number` | `0` | Buffered value (for progress indicators) |
| `step` | `number` | `1` | Step increment |
| `disabled` | `boolean` | `false` | Whether the slider is disabled |
| `name` | `string` | - | Name attribute for form submission |
| `id` | `string` | - | ID attribute |
| `thumb` | `'solid', 'outline'` | `'outline'` | Thumb style |
| `trackColor` | `string` | - | Custom track color (CSS class) |
| `bufferColor` | `string` | - | Custom buffer color (CSS class) |
| `progressColor` | `string` | - | Custom progress color (CSS class) |

### Color System

#### Variants

- **primary**: Uses theme colors (`bg-primary`, `bg-secondary`, etc.)
- **secondary**: Uses secondary theme colors 
- **black**: Uses black with opacity ratios (30%/50%/100%)
- **white**: Uses white with opacity ratios (30%/50%/100%)

#### Custom Colors

Override default colors with custom CSS classes:

```astro
<Slider 
    trackColor="bg-gray-200"
    bufferColor="bg-gray-400/50" 
    progressColor="bg-blue-500"
/>
```

### Alpine.js Integration

The component provides comprehensive Alpine.js integration:

#### State Properties

- `currentValue`: Current slider value
- `bufferedValue`: Buffer value
- `dragging`: Whether user is currently dragging
- `disabled`: Disabled state

#### Events

- `slider-change`: Fired when value changes (on mouse up)
- `slider-drag`: Fired continuously while dragging

#### Methods

- `setCurrentValue(value)`: Programmatically set the value
- `getCurrentValue()`: Get current value
- `getCurrentPercentage()`: Get current value as percentage

### Accessibility Features

- Full keyboard navigation support
- Proper ARIA attributes and roles
- Screen reader compatibility
- Focus management with visible focus indicators
- Semantic HTML with hidden native input for form compatibility

## Usage Examples

### Volume Control

```astro
<div x-data="{ 
    volume: 75, 
    isMuted: false,
    previousVolume: 75,
    toggleMute() {
        if (this.isMuted) {
            this.volume = this.previousVolume;
            this.isMuted = false;
        } else {
            this.previousVolume = this.volume;
            this.volume = 0;
            this.isMuted = true;
        }
    }
}">
    <div class="flex items-center gap-3 mb-2">
        <span class="text-sm font-medium">Volume</span>
        <button 
            x-on:click="toggleMute()"
            class="p-1 hover:bg-accent rounded"
            x-bind:class="isMuted ? 'text-muted-foreground' : ''"
        >
            <Icon :name="isMuted ? 'lucide:volume-x' : volume > 50 ? 'lucide:volume-2' : 'lucide:volume-1'" class="w-4 h-4" />
        </button>
        <span class="text-sm tabular-nums min-w-[2.5rem]" x-text="Math.round(volume) + '%'"></span>
    </div>
    
    <Slider 
        x-model="volume"
        variant="primary"
        thumb="solid"
        x-bind:disabled="isMuted"
        x-on:slider-change="isMuted = false"
    />
</div>
```
