## Overview

The Grid system provides complete grid layout functionality through two components:

- **Grid**: The container component that creates responsive grid layouts with column control and gap spacing
- **GridItems**: Simple column-oriented layout containers designed for grid content with Alpine.js integration

Grid is content-driven and only has gap properties for spacing, while GridItems provides minimal styling perfect for dynamic content with `x-for` directives.

## Grid Component API

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `columns` | `'auto', 1, 2, 3, 4, 5, 6, 12` | `'auto'` | Number of grid columns |
| `overflow` | `'auto', 'fixed', 'expand', 'wrap', 'show'` | `'auto'` | Controls overflow behavior |
| `class` | `string` | - | Additional CSS classes |
| `...alpineProps` | `any` | - | Alpine.js attributes (x-data, x-for, etc.) |

### Column Options

- **`auto`** (default): `grid-cols-[repeat(auto-fit,minmax(280px,1fr))]` - Responsive auto-fitting columns
- **`1`**: Single column layout
- **`2`**: Two columns on medium screens and up  
- **`3`**: Three columns on large screens and up
- **`4`**: Four columns on large screens and up
- **`5`**: Five columns on large screens and up
- **`6`**: Six columns on large screens and up
- **`12`**: Twelve-column layout system for complex grids

### Grid Overflow Behaviors

- **`auto`** (default): Both horizontal and vertical scrolling when needed
- **`fixed`**: Content is clipped at container boundaries
- **`expand`**: Container grows to fit all content
- **`show`**: Content can spill outside container bounds

## GridItems Component API

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `padding` | `'none', 'sm', 'md', 'lg', 'xl'` | `'md'` | Internal padding |
| `overflow` | `'auto', 'fixed', 'expand', 'wrap', 'show'` | `'expand'` | Controls overflow behavior |
| `elevation` | `'flat', 'shadow', 'elevated', 'floating', 'lifted', 'high'` | `'flat'` | Shadow/elevation level |
| `class` | `string` | - | Additional CSS classes |
| `x-for` | `string` | - | Alpine.js x-for directive |
| `x-key` | `string` | - | Alpine.js x-key directive |
| `...alpineProps` | `any` | - | Other Alpine.js attributes |

### GridItems Overflow Behaviors

- **`expand`** (default): Container grows with content
- **`auto`**: Vertical scrolling when content overflows
- **`fixed`**: Content is clipped at container boundaries
- **`show`**: Content can spill outside container bounds

### Elevation Options

- **`flat`** (default): No shadow
- **`shadow`**: Subtle shadow
- **`elevated`**: Medium shadow
- **`floating`**: Strong shadow
- **`lifted`**: Extra lift shadow
- **`high`**: Maximum elevation

### Alpine.js Integration

GridItems is designed to work seamlessly with Alpine.js `x-for`:
- Automatically extracts `x-for` and `x-key` from props
- Wraps content in `<template>` tag for Alpine.js iteration
- Passes through all other Alpine.js attributes

## Design Principles

### Grid (Container)
Grid is a **content-driven component** that:
- Only has gap property for spacing between items
- Size determined by internal elements
- Handles dynamic/data-driven content
- Default overflow is `auto` with sensible constraints
- Uses CSS Grid for precise column control
- Responsive breakpoints built into column presets

### GridItems (Content)
GridItems is a **simple layout container** that:
- Provides minimal styling with layout essentials only
- Uses `flex flex-col w-full` base classes for column layout  
- Has padding, overflow, and elevation properties for basic customization
- Custom styling should be added via `class` prop
- For rich styled content containers, use Card component instead
- Perfect for Alpine.js dynamic content with x-for directives

## When to Use Each Component

### Use Grid when:
- Creating responsive grid layouts
- Need precise column control (2, 3, 4, etc.)
- Want auto-fitting responsive behavior
- Building complex 12-column layouts
- Container-level overflow control needed

### Use GridItems when:
- Building individual grid cells with minimal styling
- Working with Alpine.js `x-for` dynamic content
- Need simple column-oriented layout
- Want elevation/shadow effects
- Content-level overflow control needed

### Use Both Together when:
- Grid provides the layout structure
- GridItems provide individual cell content with dynamic behavior
- Perfect for product grids, card layouts, and data-driven interfaces
