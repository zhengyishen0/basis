## Overview

Loading is a simple component for displaying loading states with optional spinner animations. It provides consistent styling and spacing for loading messages and can be easily customized with different sizes.

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `'sm' | 'md' | 'lg'` | `'md'` | Size of the loading component |
| `spinner` | `boolean` | `false` | Shows animated spinner icon |
| `class` | `string` | `undefined` | Additional CSS classes |

### Slots

| Slot | Description |
|------|-------------|
| default | Loading message or content |

### Size Specifications

| Size | Padding | Text Size | Description |
|------|---------|-----------|-------------|
| `sm` | `py-6` | `text-sm` | Compact loading state for small components |
| `md` | `py-12` | `text-lg` | Default size for most loading states |
| `lg` | `py-16` | `text-xl` | Large loading state for main content areas |

### Alpine.js Pass-through

The Loading component supports Alpine.js pass-through props:

```astro
<Loading 
  spinner={true}
  x-show="loading"
  x-transition
  x-on:click="handleLoadingClick()"
>
  Loading...
</Loading>
```

### Usage Examples

```astro
{/* Simple text loading */}
<Loading>Loading...</Loading>

{/* With spinner */}
<Loading spinner={true}>Fetching data...</Loading>

{/* Different sizes */}
<Loading size="sm" spinner={true}>Loading...</Loading>
<Loading size="lg" spinner={true}>Loading large dataset...</Loading>

{/* With Alpine.js */}
<Loading 
  spinner={true}
  x-show="isLoading"
  x-transition
>
  Processing request...
</Loading>

{/* Custom styling */}
<Loading 
  class="bg-accent text-accent-foreground rounded-md"
  spinner={true}
>
  Custom styled loading...
</Loading>

{/* Spinner only (no text) */}
<Loading spinner={true} />
```

### Features

- **Size Variants**: Three predefined sizes for different contexts
- **Optional Spinner**: Animated loading spinner with consistent styling
- **Alpine.js Ready**: Supports all Alpine.js directives and pass-through props
- **Flexible Content**: Accepts any content through slots
- **Responsive**: Adapts to parent container
- **Consistent Styling**: Follows design system color and spacing tokens

### Best Practices

1. **Use appropriate sizes**: Small for inline loading, medium for cards, large for page content
2. **Include descriptive text**: Help users understand what's loading
3. **Combine with Alpine.js**: Use `x-show` and `x-transition` for smooth loading states
4. **Provide alternatives**: Consider skeleton loading for better UX in some cases

### Accessibility

- Uses semantic HTML with appropriate text content
- Spinner has proper animation that respects `prefers-reduced-motion`
- Text content is readable by screen readers
- Color contrast meets WCAG standards
