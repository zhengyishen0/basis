## Overview

The Conditional component provides a clean way to handle if/else rendering logic in your templates using Alpine.js expressions.

## Use Cases

The Conditional component is ideal for:

- **Authentication States**: Show different content for logged in/out users
- **Loading States**: Display loaders vs content
- **Empty States**: Handle empty lists or missing data
- **Feature Flags**: Toggle features based on conditions
- **Permission Checks**: Show content based on user roles
- **Form Validation**: Display success/error states

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `condition` | `string` | - | **Required.** Alpine.js expression to evaluate |
| `class` | `string` | - | Additional CSS classes |

### Slots

| Slot | Description |
|------|-------------|
| `true` | Content rendered when condition evaluates to true |
| `false` | Content rendered when condition evaluates to false |

### Features

- **Alpine.js Integration**: Uses Alpine.js x-show directives for reactive rendering
- **Performance Optimized**: Only renders content when needed
- **Flexible Conditions**: Supports any valid Alpine.js expression
- **Nested Support**: Can be nested within other Conditional components
- **Clean Syntax**: Provides a declarative approach to conditional rendering

### Alpine.js Expression Examples

```javascript
// Simple boolean
condition="isVisible"

// Comparison operators
condition="count > 5"
condition="status === 'active'"
condition="user.role !== 'guest'"

// Logical operators
condition="isLoggedIn && hasPermission"
condition="theme === 'dark' || prefersDark"
condition="!loading && !error"

// Array/Object checks
condition="items.length > 0"
condition="user && user.name"
condition="errors.length === 0"

// Complex expressions
condition="(status === 'active' || status === 'pending') && !isDisabled"
```

### Best Practices

- Use descriptive variable names in your Alpine.js data
- Keep conditions simple and readable
- Consider extracting complex logic into computed properties
- Use the false slot to provide meaningful fallback content
- Combine with other Alpine.js directives for dynamic behavior
