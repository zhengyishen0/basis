## Overview

Banner is a composable notification component system built with shadcn-style composability. It supports various positions (static, fixed, sticky), different variants, and includes sub-components for structured content. Perfect for announcements, alerts, and important messages.

## API Reference

### Banner Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'default', 'secondary', 'destructive', 'success', 'warning', 'info', 'outline'` | `'default'` | Visual variant of the banner |
| `position` | `'static', 'top', 'bottom', 'sticky-top', 'sticky-bottom'` | `'static'` | Positioning behavior |
| `visible` | `boolean` | `false` | Initial visibility state |
| `visibleAfter` | `number` | `0` | Delay in milliseconds before showing |
| `bannerId` | `string` | `undefined` | Unique ID for event-based control |
| `class` | `string` | `undefined` | Additional CSS classes |

### BannerTitle Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | `undefined` | Additional CSS classes |

### BannerDescription Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | `undefined` | Additional CSS classes |

### BannerAction Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `href` | `string` | `undefined` | Renders as link if provided |
| `className` | `string` | `undefined` | Additional CSS classes |

### BannerClose Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | `undefined` | Additional CSS classes |

### Position Behaviors

| Position | Description |
|----------|-------------|
| `static` | Normal document flow |
| `top` | Fixed to top of viewport |
| `bottom` | Fixed to bottom of viewport |
| `sticky-top` | Sticks to top when scrolling |
| `sticky-bottom` | Sticks to bottom when scrolling |

### Alpine.js Events

| Event | Description | Payload |
|-------|-------------|---------|
| `show-banner` | Shows a banner | `{ bannerId: string }` |
| `hide-banner` | Hides a banner | `{ bannerId: string }` |

### Usage Examples

```astro
{/* Basic banner */}
<Banner variant="info" visible={true}>
  <BannerTitle>Simple message</BannerTitle>
</Banner>

{/* Rich banner */}
<Banner variant="warning" visible={true}>
  <Icon name="lucide:alert-triangle" />
  <div class="flex-1">
    <BannerTitle>Important Update</BannerTitle>
    <BannerDescription>
      System maintenance scheduled for tonight.
    </BannerDescription>
  </div>
  <BannerAction href="/maintenance">
    Learn More
  </BannerAction>
  <BannerClose />
</Banner>

{/* Event-controlled banner */}
<Banner 
  variant="success" 
  visible={false}
  bannerId="success-banner"
>
  <BannerTitle>Operation Complete</BannerTitle>
</Banner>

{/* Delayed banner */}
<Banner 
  variant="info"
  visible={false}
  visibleAfter={3000}
>
  <BannerTitle>Welcome back!</BannerTitle>
</Banner>

{/* Fixed position */}
<Banner 
  variant="destructive" 
  position="top"
  visible={true}
>
  <BannerTitle>Critical Alert</BannerTitle>
  <BannerClose />
</Banner>
```

### Features

- **Composable Architecture**: Mix and match sub-components as needed
- **Multiple Variants**: Full shadcn-style color system support
- **Flexible Positioning**: Static, fixed, and sticky positioning options
- **Alpine.js Integration**: Event-based show/hide with smooth transitions
- **Delayed Visibility**: Built-in timer for delayed banner appearance
- **Responsive Design**: Adapts to different screen sizes
- **Accessible**: Proper ARIA attributes and keyboard support

### Best Practices

1. **Use appropriate variants**: Match the banner color to the message importance
2. **Include clear actions**: Provide next steps when possible
3. **Position strategically**: Use fixed/sticky positions for critical messages only
4. **Provide dismissal**: Always include a close button for non-critical banners
5. **Control timing**: Use delayed visibility for onboarding or progressive disclosure

### Accessibility

- Proper semantic HTML structure
- ARIA labels for close buttons
- Keyboard navigation support
- Color contrast compliance
- Screen reader friendly content structure
