## Overview

The Dialog system provides a complete set of components for creating modal dialogs, confirmation prompts, and overlay content. Built with Alpine.js for smooth animations and proper focus management, the system supports keyboard shortcuts, auto-focus, and flexible positioning variants.

### Component Architecture

The dialog system consists of these core components:

- **Dialog**: Main container with state management and keyboard handling
- **DialogContent**: Content wrapper with backdrop and positioning variants
- **DialogHeader**: Header section with optional close button
- **DialogFooter**: Footer section with flexible button layouts
- **DialogTrigger**: Trigger element that opens the dialog

## Component API Reference

### Dialog (Main Container)

The root Dialog component manages state, keyboard shortcuts, and focus handling.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `closeable` | `boolean` | `true` | Whether dialog can be closed with Escape key |
| `lockScroll` | `boolean` | `true` | Prevent body scrolling when dialog is open |
| `keyboardShortcut` | `string` | - | Keyboard shortcut to open dialog (e.g., "cmd+k", "ctrl+space") |
| `autoFocus` | `boolean` | `false` | Auto-focus first input/button when opened |
| `focusSelector` | `string` | `'input, button, [tabindex]:not([tabindex="-1"])'` | CSS selector for auto-focus element |
| `className` | `string` | - | Additional CSS classes |

**Alpine.js State:**

- `open`: `boolean` - Whether dialog is currently open
- `toggle()`: Function to toggle dialog state
- `close()`: Function to close dialog and reset form fields

### DialogContent

Content wrapper with backdrop, animations, and positioning variants.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'center' \| 'left' \| 'right' \| 'top' \| 'bottom' \| 'fullscreen' \| 'positioned'` | `'center'` | Dialog positioning |
| `backdrop` | `'dark' \| 'light' \| 'blur' \| 'none'` | `'dark'` | Backdrop style |
| `className` | `string` | - | Additional CSS classes |

**Variant Details:**

- `center`: Centered modal dialog
- `left`: Slides in from left side (sidebar style)
- `right`: Slides in from right side (sidebar style) 
- `top`: Slides down from top (banner style)
- `bottom`: Slides up from bottom (sheet style)
- `fullscreen`: Covers entire viewport
- `positioned`: Absolute positioning for custom placement

### DialogHeader

Header section with optional close button and title content.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `closeable` | `boolean` | `true` | Show close button in top-right corner |
| `className` | `string` | - | Additional CSS classes |

**Default Styling:**
- Flex layout with title content on left
- Close button positioned absolutely in top-right
- Minimum height for consistent spacing

### DialogFooter

Footer section with flexible button layouts and spacing.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `justify` | `'start' \| 'end' \| 'center' \| 'between'` | `'end'` | Button alignment |
| `gap` | `'sm' \| 'md' \| 'lg'` | `'sm'` | Spacing between buttons |
| `className` | `string` | - | Additional CSS classes |

**Layout Options:**
- `start`: Buttons aligned to left
- `end`: Buttons aligned to right (default)
- `center`: Buttons centered
- `between`: Buttons spread apart with space between

### DialogTrigger

Trigger element that opens the dialog when clicked.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'default' \| 'button' \| 'outline' \| 'ghost'` | `'default'` | Visual style |
| `className` | `string` | - | Additional CSS classes |

**Variant Styles:**
- `default`: Unstyled (inherit parent styling)
- `button`: Primary button appearance
- `outline`: Outlined button style
- `ghost`: Subtle hover effects only

## Alpine.js Integration

The Dialog system is designed to work seamlessly with Alpine.js. Here are common integration patterns:

### Basic State Management

```javascript
// Simple dialog data
x-data="{
    formData: {
        name: '',
        email: '',
        role: 'user'
    },
    
    submitForm() {
        // Handle form submission
        console.log('Submitting:', this.formData);
        // Close dialog after successful submission
        this.$dispatch('close-dialog');
    },
    
    resetForm() {
        this.formData = { name: '', email: '', role: 'user' };
    }
}"
```

### Dynamic Content

```javascript
// Dynamic dialog content
x-data="{
    selectedItem: null,
    
    showDetails(item) {
        this.selectedItem = item;
        this.$dispatch('open-dialog');
    },
    
    deleteItem(id) {
        // Delete logic here
        this.selectedItem = null;
        this.$dispatch('close-dialog');
    }
}"
```

### Form Validation

```javascript
// Form validation
x-data="{
    form: { name: '', email: '' },
    errors: {},
    
    validateForm() {
        this.errors = {};
        
        if (!this.form.name) {
            this.errors.name = 'Name is required';
        }
        
        if (!this.form.email) {
            this.errors.email = 'Email is required';
        } else if (!this.isValidEmail(this.form.email)) {
            this.errors.email = 'Invalid email format';
        }
        
        return Object.keys(this.errors).length === 0;
    },
    
    submitForm() {
        if (this.validateForm()) {
            // Submit form
            this.$dispatch('close-dialog');
        }
    },
    
    isValidEmail(email) {
        return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
    }
}"
```

## Accessibility

The Dialog system includes comprehensive accessibility features:

### Keyboard Navigation
- **Escape Key**: Closes dialog (unless `closeable={false}`)
- **Tab Navigation**: Proper focus management within dialog
- **Keyboard Shortcuts**: Global shortcuts work from anywhere on page
- **Auto Focus**: Focus first interactive element when opened

### ARIA Attributes
- `role="dialog"`: Identifies content as modal dialog
- `aria-modal="true"`: Indicates modal behavior
- `aria-expanded`: Shows trigger state
- `aria-controls`: Links trigger to dialog content

### Focus Management
- Traps focus within dialog when open
- Returns focus to trigger when closed
- Auto-focuses first input/button if enabled
- Custom focus selector support

### Screen Reader Support
- Semantic HTML structure
- Descriptive labels and instructions
- Status announcements for state changes

## Best Practices

### Performance
- Use `teleport` for dialog content to prevent z-index issues
- Implement proper cleanup of event listeners
- Consider lazy loading for complex dialog content
- Use debounced input handlers for search dialogs

### User Experience
- Provide clear visual feedback for all interactions
- Use appropriate dialog variants for different use cases
- Include helpful error states and validation messages
- Maintain consistent button layouts and terminology

### Accessibility
- Always include descriptive dialog titles
- Provide keyboard shortcuts for frequently used dialogs
- Test with screen readers and keyboard-only navigation
- Use semantic HTML and proper ARIA attributes

### Development
- Keep dialog content focused and concise
- Use consistent button placement (cancel left, primary right)
- Implement proper form validation and error handling
- Test dialog behavior across different screen sizes

The Dialog system provides a solid foundation for creating accessible, performant modal experiences in your Astro applications with full Alpine.js integration.
