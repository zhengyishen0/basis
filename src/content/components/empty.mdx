## Overview

Empty is a minimalist component for displaying empty states with consistent styling and spacing. It's perfect for showing messages when there's no content to display, such as empty lists, search results, or data tables.

                clearItems() {
                    this.items = [];
                },
                get filteredItems() {

                    x-on:click="clearItems()"

                </button>
                
                <input

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `'sm' | 'md' | 'lg'` | `'md'` | Size of the empty state component |
| `class` | `string` | `undefined` | Additional CSS classes |

### Slots

| Slot | Description |
|------|-------------|
| default | Empty state message or content |

### Size Specifications

| Size | Padding | Text Size | Line Height | Use Case |
|------|---------|-----------|-------------|----------|
| `sm` | `py-6` | `text-sm` | `normal` | Compact spaces, table rows, small lists |
| `md` | `py-12` | `text-lg` | `leading-relaxed` | Standard containers, cards, sections |
| `lg` | `py-16` | `text-xl` | `leading-relaxed` | Full page states, main content areas |

### Alpine.js Pass-through

The Empty component supports all Alpine.js directives:

```astro
<Empty 
  size="md"
  x-show="items.length === 0"
  x-transition
  x-on:click="showHelp = true"
>
  Empty state content
</Empty>
```

### Features

- **Flexible Sizing**: Three responsive size variants for different contexts
- **Clean Design**: Minimal styling that works with any design system
- **Alpine.js Ready**: Full support for reactive data binding and transitions
- **Semantic HTML**: Uses proper text styling and spacing
- **Customizable**: Easy to extend with additional classes and content
- **Accessible**: Proper color contrast and readable text

### Best Practices

1. **Choose appropriate sizes**: Small for inline/compact spaces, medium for standard containers, large for full page states
2. **Provide helpful messages**: Explain what the user can do or what they should expect
3. **Use with loading states**: Show empty states only after data has loaded
4. **Add calls to action**: Consider including buttons or links in larger empty states
5. **Be contextual**: Tailor the message to the specific feature or section
