## Overview

NavigationMenu is a comprehensive navigation system supporting complex dropdown menus with rich content layouts. The component system includes hover and keyboard interaction management, multiple layout types (list, grid, feature), and seamless Alpine.js integration for dynamic menu generation.

The menu supports three content types: simple links, list layouts for basic navigation, grid layouts for organized content sections, and feature layouts combining prominent featured content with supporting links. All interactions are managed through Alpine.js with proper focus management, keyboard navigation, and accessibility features.

## API Reference

### NavigationMenu

Main container for navigation menu with Alpine.js state management.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `orientation` | `'horizontal' | 'vertical'` | `'horizontal'` | Menu layout orientation |
| `delayDuration` | `number` | `200` | Delay before closing menu on mouse leave (ms) |
| `skipDelayDuration` | `number` | `300` | Skip delay when quickly switching menus (ms) |

#### Alpine.js State

| Property | Type | Description |
|----------|------|-------------|
| `activeMenu` | `string` | ID of currently active menu |
| `openMenus` | `string[]` | Array of open menu IDs |
| `keyboardActive` | `boolean` | Whether keyboard navigation is active |

#### Alpine.js Methods

| Method | Parameters | Description |
|--------|------------|-------------|
| `openMenu(menuId)` | `string` | Open specific menu |
| `closeMenu(menuId)` | `string` | Close specific menu |
| `closeAllMenus()` | - | Close all open menus |
| `scheduleClose(menuId)` | `string` | Schedule menu close with delay |
| `cancelClose()` | - | Cancel scheduled close |

### NavigationMenuList

Container for navigation menu items.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |

### NavigationMenuItem

Individual menu item container supporting both links and dropdown menus.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | Unique identifier for the menu item |
| `className` | `string` | - | Additional CSS classes |
| `item` | `object` | - | Dynamic item data for Alpine.js rendering |

#### Slots

| Slot | Description |
|------|-------------|
| default | Static content when not using dynamic rendering |

### NavigationMenuItems

Template component for rendering menu items dynamically with x-for.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `x-for` | `string` | - | Alpine.js x-for expression |
| `x-key` | `string` | `'item.id'` | Alpine.js x-key expression |

#### Slots

| Slot | Description |
|------|-------------|
| `link` | Content for simple link items |
| `trigger` | Content for dropdown menu triggers |
| `content` | Dropdown menu content |

| `className` | `string` | - | Additional CSS classes |
| `disabled` | `boolean` | `false` | Whether the trigger is disabled |

#### Slots

| Slot | Description |
|------|-------------|
| default | Trigger button content |

## Data Structure

For dynamic navigation menus, use this data structure:

```typescript
interface NavigationMenuItemData {
  id: string;                    // Unique identifier
  label: string;                 // Display text
  type: 'link' | 'list' | 'grid' | 'feature'; // Item type
  href?: string;                 // URL for link type
  disabled?: boolean;            // Whether item is disabled
  links?: Array<{               // Sub-links for dropdown types
    id: string;
    title: string;
    url: string;
    description?: string;
  }>;
  gridColumns?: number;          // Columns for grid layout
  featured?: {                   // Featured content for feature type
    title: string;
    description: string;
    href: string;
    image?: string;
  };
  layout?: {                     // Layout configuration
    minWidth?: string;
    gridColumns?: number;
  };
}
```

## Alpine.js Integration

### Hover Behavior

The menu system manages hover states automatically:

- **Hover delay**: Configurable delay before opening/closing menus
- **Quick switching**: Skip delays when rapidly switching between menus  
- **Mouse leave**: Graceful closing with configurable delay

### Keyboard Navigation

Full keyboard support with proper focus management:

- **Arrow keys**: Navigate between menu items
- **Enter/Space**: Open/close dropdown menus
- **Escape**: Close all menus and return focus
- **Tab**: Standard tab order through interactive elements

### State Management

All menu state is managed through Alpine.js:

```javascript
// Access current state
activeMenu        // Currently open menu ID
openMenus        // Array of open menu IDs
keyboardActive   // Whether using keyboard navigation

// Control menus programmatically
openMenu('menu-id')
closeMenu('menu-id')
closeAllMenus()
```

## Accessibility

- **ARIA attributes**: Proper `aria-expanded`, `aria-current` states
- **Keyboard navigation**: Full keyboard support with logical focus flow
- **Screen reader support**: Semantic markup with navigation landmarks
- **Focus management**: Proper focus trapping and restoration
- **High contrast**: Supports high contrast mode and custom themes

## Layout Types

### List Layout
Single column layout for simple navigation lists. Best for text-heavy menus with minimal visual complexity.

### Grid Layout  
Multi-column grid layout for organized content sections. Configurable column count with responsive behavior.

### Feature Layout
Combines a prominent featured section with supporting links. Ideal for highlighting key content or new features.

## Best Practices

1. **Consistent structure**: Maintain consistent menu depth and organization
2. **Reasonable content**: Avoid overcrowding dropdown menus 
3. **Clear labels**: Use descriptive, actionable menu labels
4. **Logical grouping**: Group related items together
5. **Mobile consideration**: Test navigation on touch devices
6. **Performance**: Implement lazy loading for large menu datasets
7. **Visual hierarchy**: Use proper typography and spacing scales
