## Overview

The RadioGroup component provides a composable radio button group with support for static and dynamic options. Features flexible layouts (vertical/horizontal), Alpine.js state management, and a clean composable API similar to modern UI libraries.

## Components

### RadioGroup

The root component that manages state and provides context to child components.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `name` | `string` | - | **Required.** Name for the radio group |
| `selectedOption` | `string` | `''` | Initially selected option value |
| `disabled` | `boolean` | `false` | Disable the entire group |
| `orientation` | `'vertical', 'horizontal'` | `'vertical'` | Layout direction |
| `maxWidth` | `'sm', 'md', 'lg', 'xl', 'full'` | `'full'` | Maximum width constraint |
| `centered` | `boolean` | `false` | Center the group horizontally |
| `overflow` | `'fixed', 'scroll', 'hidden'` | `'fixed'` | Overflow behavior |
| `class` | `string` | - | Additional CSS classes |

### RadioGroupItem

Individual radio option container with visual styling and interaction handling.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | **Required.** Value for this radio option |
| `disabled` | `boolean` | `false` | Disable this specific option |
| `layout` | `'vertical', 'horizontal'` | `'vertical'` | Layout direction for content |
| `size` | `'sm', 'md', 'lg'` | `'md'` | Size of the option container |
| `option` | `any` | - | Dynamic option data (used with RadioGroupItems) |
| `class` | `string` | - | Additional CSS classes |

#### Slots

| Slot | Description |
|------|-------------|
| Default | Content area for RadioGroupLabel and RadioGroupDescription |

### RadioGroupItems

Template component for rendering dynamic options using Alpine.js `x-for` directive.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `x-for` | `string` | - | **Required.** Alpine.js x-for expression |
| `x-key` | `string` | `'option.value'` | Key for x-for iteration |
| All RadioGroupItem props | - | - | Passed through to each rendered item |

### RadioGroupLabel

Label component for radio options with multiple visual variants.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `text` | `string` | - | Static text content |
| `variant` | `'default', 'emphasized', 'subtle'` | `'default'` | Visual style variant |
| `size` | `'sm', 'md', 'lg'` | `'md'` | Text size |
| `overflow` | `'fixed', 'scroll', 'hidden'` | `'fixed'` | Text overflow behavior |
| `class` | `string` | - | Additional CSS classes |

#### Slots

| Slot | Description |
|------|-------------|
| Default | Label content (can be combined with `text` prop) |

### RadioGroupDescription

Description component for providing additional context about radio options.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `text` | `string` | - | Static text content |
| `variant` | `'default', 'emphasized', 'subtle'` | `'default'` | Visual style variant |
| `size` | `'xs', 'sm', 'md'` | `'sm'` | Text size |
| `overflow` | `'fixed', 'scroll', 'hidden'` | `'fixed'` | Text overflow behavior |
| `class` | `string` | - | Additional CSS classes |

#### Slots

| Slot | Description |
|------|-------------|
| Default | Description content (can be combined with `text` prop) |

## Alpine.js Integration

The RadioGroup automatically integrates with Alpine.js and provides reactive state management:

- Use `x-model` on the RadioGroup for two-way data binding
- Access `selectedOption` reactive property
- Use `setSelectedOption(value)` method to programmatically change selection
- Component automatically handles disabled states and validation

### Advanced Alpine.js Usage

```astro
<div x-data="{
    selectedPlan: 'basic',
    plans: [
        { 
            value: 'basic', 
            label: 'Basic', 
            description: 'Essential features for individuals',
            price: 9,
            disabled: false 
        },
        { 
            value: 'pro', 
            label: 'Professional', 
            description: 'Advanced features for teams',
            price: 19,
            disabled: false 
        },
        { 
            value: 'enterprise', 
            label: 'Enterprise', 
            description: 'Full-featured solution',
            price: null,
            disabled: true 
        }
    ],
    get selectedPlanData() {
        return this.plans.find(p => p.value === this.selectedPlan);
    }
}">
    <RadioGroup name="plan" x-model="selectedPlan">
        <RadioGroupItems x-for="plan in plans" x-key="plan.value">
            <div class="flex justify-between items-start">
                <div>
                    <RadioGroupLabel x-text="plan.label" variant="emphasized" />
                    <RadioGroupDescription x-text="plan.description" />
                </div>
                <div class="text-right" x-show="plan.price">
                    <span class="text-lg font-semibold" x-text="'$' + plan.price"></span>
                    <div class="text-sm text-muted-foreground">/month</div>
                </div>
            </div>
        </RadioGroupItems>
    </RadioGroup>
    
    <div class="mt-6 p-4 bg-muted rounded-lg" x-show="selectedPlanData">
        <h4 class="font-medium">Selected Plan:</h4>
        <p x-text="selectedPlanData?.label"></p>
        <p x-text="selectedPlanData?.description" class="text-sm text-muted-foreground"></p>
    </div>
</div>
```
