## States

The RadioIndicator supports two primary visual states:

### Unchecked
- Default state with empty radio circle
- Border visible, no inner dot

### Checked
- Filled state with inner dot indicator
- Smooth scale animation on selection
- Slight scale increase (105%) when selected

### Disabled
- Reduced opacity and no interaction
- Maintains visual state but prevents user input

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `checked` | `boolean` | `false` | Checked state of the radio button |
| `disabled` | `boolean` | `false` | Disabled state |
| `state` | `'default' \, 'checked' \, 'disabled'` | `'default'` | Visual state variant |
| `name` | `string` | - | Form input name attribute |
| `value` | `string` | - | Form input value |
| `class` | `string` | - | Additional CSS classes |

### Alpine.js Props

When using with Alpine.js, you can pass reactive expressions:

| Prop | Type | Description |
|------|------|-------------|
| `:checked` | `string` | Alpine.js expression for checked state |
| `:disabled` | `string` | Alpine.js expression for disabled state |

### Features

- **Smooth Animations**: Scale and opacity transitions for state changes
- **Alpine.js Integration**: Reactive state management with Alpine expressions
- **Form Compatible**: Works with form submissions and validation
- **Accessible**: Proper ARIA attributes and keyboard navigation
- **Customizable**: CVA variants and custom CSS classes
- **Touch Friendly**: Adequate touch target size for mobile devices

### Animation Details

The component includes these smooth transitions:
- **Scale Animation**: 1.0 to 1.05 scale when selected
- **Opacity Transition**: Fade in/out for inner dot
- **Duration**: 300ms ease-out timing
- **Transform GPU**: Hardware-accelerated animations

### Best Practices

- Always use with proper form labels for accessibility
- Ensure only one radio button can be selected in a group
- Provide clear visual feedback for the selected state
- Use descriptive labels and help text when needed
- Group related options logically
- Consider using card layouts for complex options

### Accessibility

- Includes proper `role` and `aria-*` attributes
- Works with screen readers and keyboard navigation
- Supports arrow key navigation between radio options
- Maintains focus management in radio groups
- Compatible with form validation and error states
