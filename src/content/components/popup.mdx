## Overview

The Popup system provides lightweight components for creating tooltips, popovers, and contextual content that appears on demand. Built with Alpine.js for smooth interactions, the system supports multiple trigger methods (click, hover, focus, right-click), automatic positioning, and flexible content styling.

### Component Architecture

The popup system consists of these core components:

- **Popup**: Main container with state management and trigger handling
- **PopupContent**: Content wrapper with positioning variants and themes
- **PopupTrigger**: Trigger element with multiple interaction options

## Component API Reference

### Popup (Main Container)

The root Popup component manages state, trigger handling, and popup positioning.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `trigger` | `'click' \| 'hover' \| 'focus' \| 'rightclick' \| 'manual'` | `'click'` | How the popup is triggered |
| `dismiss` | `'auto' \| 'manual'` | `'manual'` | Auto-dismiss behavior |
| `delay` | `number` | `0` | Delay in ms before showing popup |
| `className` | `string` | - | Additional CSS classes |

**Alpine.js State:**

- `open`: `boolean` - Whether popup is currently open
- `trigger`: `string` - Current trigger method
- `popupId`: `string` - Unique identifier for this popup
- `toggle()`: Function to toggle popup state
- `close()`: Function to close popup

**Trigger Behaviors:**

- `click`: Opens/closes on click, closes on outside click
- `hover`: Opens on mouseenter, closes on mouseleave  
- `focus`: Opens on focus, closes on blur
- `rightclick`: Opens on right-click (prevents context menu)
- `manual`: Only controlled via Alpine.js methods

### PopupContent

Content wrapper with positioning, themes, and transition animations.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `theme` | `'default' \| 'dark' \| 'minimal'` | `'default'` | Visual theme |
| `placement` | `'top' \| 'bottom' \| 'left' \| 'right'` | `'bottom'` | Popup positioning |
| `role` | `string` | `'dialog'` | ARIA role attribute |
| `className` | `string` | - | Additional CSS classes |

**Theme Variations:**

- `default`: Standard popover with border and shadow
- `dark`: Dark background with white text
- `minimal`: Clean white background with subtle shadow

**Placement Options:**

- `top`: Appears above trigger (bottom margin)
- `bottom`: Appears below trigger (top margin)
- `left`: Appears to left of trigger (right margin, vertically centered)
- `right`: Appears to right of trigger (left margin, vertically centered)

### PopupTrigger

Trigger element with styling variants and interaction handling.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'default' \| 'button' \| 'ghost' \| 'link'` | `'default'` | Visual style |
| `className` | `string` | - | Additional CSS classes |

**Variant Styles:**

- `default`: Unstyled (inherit parent styling)
- `button`: Button-like appearance with padding and hover effects
- `ghost`: Subtle hover effects only
- `link`: Underlined link styling

## Alpine.js Integration

The Popup system integrates seamlessly with Alpine.js for dynamic content and state management:

### Basic Integration

```javascript
// Simple popup state
x-data="{
    showDetails: false,
    selectedItem: null,
    
    showItemDetails(item) {
        this.selectedItem = item;
        this.showDetails = true;
    }
}"
```

### Dynamic Content

```javascript
// Dynamic popup content based on data
x-data="{
    users: [
        { id: 1, name: 'John Doe', role: 'Admin' },
        { id: 2, name: 'Jane Smith', role: 'User' }
    ],
    selectedUser: null,
    
    selectUser(user) {
        this.selectedUser = user;
        // Popup will show selected user data
    }
}"
```

### Form Integration

```javascript
// Form handling within popup
x-data="{
    formData: { name: '', email: '', message: '' },
    isSubmitting: false,
    
    async submitForm() {
        this.isSubmitting = true;
        try {
            // Submit form data
            await fetch('/api/contact', {
                method: 'POST',
                body: JSON.stringify(this.formData)
            });
            // Reset and close
            this.formData = { name: '', email: '', message: '' };
            this.$dispatch('close-popup');
        } catch (error) {
            console.error('Submission failed:', error);
        } finally {
            this.isSubmitting = false;
        }
    }
}"
```

## Accessibility

The Popup system includes built-in accessibility features:

### Keyboard Support
- **Escape Key**: Closes popup when open
- **Focus Management**: Maintains proper focus handling
- **Tab Navigation**: Works within popup content

### ARIA Attributes
- `role="dialog"`: Identifies popup content
- `aria-expanded`: Shows trigger state
- `aria-controls`: Links trigger to content

### Screen Reader Support
- Semantic HTML structure
- Descriptive content and labels
- Proper focus announcements

## Best Practices

### Performance
- Use `trigger="hover"` sparingly for better mobile experience
- Implement delays for hover triggers to prevent accidental activation
- Keep popup content lightweight for smooth animations

### User Experience
- Use appropriate trigger methods based on content type
- Provide clear visual feedback for interactive elements
- Keep popup content concise and focused
- Position popups to avoid viewport edges

### Development
- Use semantic HTML within popup content
- Test keyboard navigation and screen reader compatibility
- Implement proper error handling for dynamic content
- Consider mobile responsiveness for popup positioning

The Popup system provides a flexible foundation for creating tooltips, popovers, and contextual content that enhances user experience without overwhelming the interface.
