## Overview

Tabs organize content into multiple sections and allow users to navigate between them. Each tab panel contains content that is independent of other panels.

## Basic Usage

<Tabs defaultValue="tab1" class="w-full">
    <TabsList>
        <TabsTrigger value="tab1">Account</TabsTrigger>
        <TabsTrigger value="tab2">Password</TabsTrigger>
        <TabsTrigger value="tab3">Settings</TabsTrigger>
    </TabsList>
    <TabsContent value="tab1">
        <div class="p-4">
            <h3 class="font-semibold mb-2">Account Information</h3>
            <p>Manage your account details and preferences here.</p>
        </div>
    </TabsContent>
    <TabsContent value="tab2">
        <div class="p-4">
            <h3 class="font-semibold mb-2">Password Settings</h3>
            <p>Update your password and security settings.</p>
        </div>
    </TabsContent>
    <TabsContent value="tab3">
        <div class="p-4">
            <h3 class="font-semibold mb-2">General Settings</h3>
            <p>Configure your application preferences.</p>
        </div>
    </TabsContent>
</Tabs>

## Alpine.js Integration

The Tabs component uses Alpine.js for state management:

<Code 
    code={'<div x-data="{ activeTab: \'tab1\' }">\n    <TabsList>\n        <TabsTrigger @click="activeTab = \'tab1\'" :class="activeTab === \'tab1\' && \'active\'">\n            Tab 1\n        </TabsTrigger>\n    </TabsList>\n</div>'}
    language="astro" 
/>

## API Reference

### Props

| Component | Props | Description |
|-----------|-------|-------------|
| `Tabs` | `defaultValue`, `class` | Container with default active tab |
| `TabsList` | `class` | Container for tab triggers |
| `TabsTrigger` | `value`, `class` | Individual tab button |
| `TabsContent` | `value`, `class` | Content panel for each tab |
