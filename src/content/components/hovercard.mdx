## API Reference

### HoverCard Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `delay` | `number` | `200` | Delay in milliseconds before hover card appears |
| `className` | `string` | - | Additional CSS classes |

### PopupTrigger Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'default' \| 'button' \| 'ghost' \| 'link'` | `'default'` | Visual style of the trigger |
| `className` | `string` | - | Additional CSS classes |

### PopupContent Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `theme` | `'default' \| 'dark' \| 'minimal'` | `'default'` | Visual theme of the hover card |
| `placement` | `'top' \| 'bottom' \| 'left' \| 'right'` | `'bottom'` | Position relative to trigger |
| `className` | `string` | - | Additional CSS classes |
| `role` | `string` | `'dialog'` | ARIA role for accessibility |

### Slots

| Component | Slot | Description |
|-----------|------|-------------|
| PopupTrigger | `default` | Content that triggers the hover card |
| PopupContent | `default` | Rich hover card content |

### Alpine.js Integration

The hover card automatically manages state using Alpine.js:

```javascript
// Automatically available in x-data scope:
{
  open: false,           // Hover card visibility state
  trigger: 'hover',      // Trigger type (always 'hover' for hover cards)
  popupId: 'popup-xxx',  // Unique ID for accessibility
  toggle() { ... },      // Toggle hover card visibility
  close() { ... }        // Close hover card
}
```

### Accessibility

The hover card component includes built-in accessibility features:

- ARIA attributes for screen readers
- Keyboard navigation support (Escape to close)
- Focus management
- Semantic HTML structure
- Proper role attribution

### Styling Guidelines

- Use appropriate width classes (`w-64`, `w-80`, `w-96`) for content
- Include proper spacing with gap utilities
- Use semantic HTML elements within content
- Apply consistent padding and margins
- Consider content hierarchy with typography scales
- Use theme variants for different contexts

### Performance Considerations

- Content is only rendered when hovered (lazy loading)
- Smooth animations with Alpine.js transitions
- Minimal DOM impact when not active
- Efficient event handling with Alpine.js
- Small delay prevents accidental triggers
