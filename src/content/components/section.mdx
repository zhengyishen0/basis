## Overview

The Section component is a semantic wrapper that provides proper HTML structure for page sections. It supports different HTML tags (section, div, article, aside) and alignment options (left, center, right) for flexible content organization.

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `tag` | `'section', 'div', 'article', 'aside'` | `'section'` | HTML element to render |
| `align` | `'left', 'center', 'right'` | `'center'` | Content alignment |
| `class` | `string` | - | Additional CSS classes |
| `...alpineProps` | `any` | - | Alpine.js attributes |

### HTML Tag Options

- **`section`** (default): Semantic section element for distinct page sections
- **`article`**: For standalone content like blog posts, news articles
- **`aside`**: For complementary content like sidebars, callouts
- **`div`**: Generic container when no semantic meaning is needed

### Alignment Options

- **`left`**: `items-start` - Content aligned to left
- **`center`** (default): `items-center` - Content centered
- **`right`**: `items-end` - Content aligned to right

### Base Classes

The Section component applies alignment classes but no other styling by default, making it a pure semantic wrapper.

## Design Principles

Section is a **semantic wrapper component** that:
- Provides proper HTML structure with appropriate semantic tags
- Supports different content alignments through CSS flexbox
- Remains unstyled by default to allow full customization
- Works with any content layout components (Column, Grid, etc.)
- Maintains accessibility through proper semantic structure

## Common Use Cases

- **Landing Page Sections**: Hero, features, testimonials sections
- **Article Content**: Blog posts, documentation, news articles
- **Sidebar Content**: Navigation, related links, supplementary info
- **Generic Containers**: When grouping content without semantic meaning
- **Component Wrappers**: Providing structure around complex layouts

## Semantic HTML Guidelines

| Tag | Use For | Examples |
|-----|---------|----------|
| `<section>` | Distinct page sections | Hero, features, about, contact |
| `<article>` | Standalone content | Blog posts, news articles, documentation |
| `<aside>` | Supplementary content | Sidebars, related links, callouts |
| `<div>` | Generic grouping | Layout containers, styling wrappers |

## Accessibility Benefits

- **Screen Readers**: Proper semantic structure helps navigation
- **Document Outline**: Creates logical page structure
- **Keyboard Navigation**: Semantic landmarks for better UX
- **SEO**: Search engines understand content structure better
