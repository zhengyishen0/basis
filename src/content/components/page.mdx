## Overview

The Page component is a full-screen page wrapper that provides the foundation for page layouts. It uses the semantic `<main>` element, has `min-h-screen` for full height, and offers responsive max-width options with centered content.

## API Reference

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `maxWidth` | `'full' | 'narrow' | 'container' | 'wide'` | `'full'` | Maximum width constraint |
| `overflow` | `'auto' | 'fixed'` | `'auto'` | Overflow behavior |
| `class` | `string` | - | Additional CSS classes |
| `...alpineProps` | `any` | - | Alpine.js attributes |

### Max-Width Options

- **`full`** (default): `w-full` - No width constraints
- **`narrow`**: `max-w-2xl mx-auto` - Optimal for reading content (~768px)
- **`container`**: `max-w-7xl mx-auto` - App dashboards and interfaces (~1280px)
- **`wide`**: `max-w-screen-xl mx-auto` - Marketing pages and wide layouts (~1536px)

### Overflow Options

- **`auto`** (default): `overflow-auto` - Natural scrolling behavior
- **`fixed`**: `overflow-hidden` - For full-screen applications without scroll

### Base Classes

- **`min-h-screen`**: Ensures page takes full viewport height
- **Semantic `<main>` element**: Proper HTML semantics for page content

## Design Principles

Page is a **foundational layout component** that:
- Provides semantic structure with `<main>` element
- Ensures full viewport height with `min-h-screen`
- Offers responsive max-width options for different content types
- Centers content automatically with `mx-auto`
- Handles overflow behavior appropriately
- Works as the top-level wrapper for page layouts

## Common Use Cases

- **Blog/Article Pages**: Use `narrow` for optimal reading width
- **Application Dashboards**: Use `container` for balanced space usage
- **Marketing Landing Pages**: Use `wide` for maximum content display
- **Full-Screen Apps**: Use `full` width with `fixed` overflow
- **Documentation**: Use `narrow` or `container` for readability

## Layout Patterns

| Layout Type | Max-Width | Use Case | Example |
|-------------|-----------|----------|---------|
| Reading | `narrow` | Blogs, articles, docs | News sites, documentation |
| Application | `container` | Dashboards, forms | Admin panels, SaaS apps |
| Marketing | `wide` | Landing pages | Product showcases, portfolios |
| Full-Screen | `full` | Apps, games | IDEs, design tools, games |
