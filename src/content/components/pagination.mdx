## Overview

Pagination components provide a complete solution for navigating through large datasets with intelligent page calculation, configurable visible pages, and seamless Alpine.js integration. The system includes a main `Pagination` container with built-in state management, composable sub-components (`PaginationItem`, `PaginationLink`, `PaginationNext`, `PaginationPrevious`), and automatic ellipsis handling for large page ranges.

The components feature smart page visibility algorithms, customizable information display, and full keyboard accessibility. All state management is handled internally with Alpine.js, making integration straightforward while maintaining complete control over styling and behavior.

## API Reference

### Pagination

Main pagination container with built-in Alpine.js state management.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `currentPage` | `number` | `1` | Current active page number |
| `totalPages` | `number` | `1` | Total number of pages |
| `totalItems` | `number` | `0` | Total number of items in dataset |
| `pageSize` | `number` | `10` | Number of items per page |
| `maxVisible` | `number` | `5` | Maximum visible page numbers (deprecated, use maxElements) |
| `maxElements` | `number` | `7` | Maximum total elements including ellipsis |
| `showInfo` | `boolean` | `true` | Whether to show info slot content |
| `showFirstLast` | `boolean` | `true` | Whether to show first/last page navigation |
| `class` | `string` | - | Additional CSS classes |

#### Slots

| Slot | Description |
|------|-------------|
| default | Pagination controls and navigation elements |
| `info` | Custom information display (item counts, etc.) |

#### Alpine.js State

The Pagination component provides the following reactive properties:

| Property | Type | Description |
|----------|------|-------------|
| `visiblePages` | `Array of PageObject` | Array of page objects for rendering (PageObject has number, current, and isEllipsis properties) |
| `hasPrevious` | `boolean` | Whether previous page navigation is available |
| `hasNext` | `boolean` | Whether next page navigation is available |
| `startItem` | `number` | First item number on current page |
| `endItem` | `number` | Last item number on current page |

#### Alpine.js Methods

| Method | Parameters | Description |
|--------|------------|-------------|
| `goToPage(page)` | `number` | Navigate to specific page |
| `nextPage()` | - | Navigate to next page |
| `previousPage()` | - | Navigate to previous page |

### PaginationItem

Template component for rendering page items dynamically.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `class` | `string` | - | Additional CSS classes |

#### Slots

| Slot | Description |
|------|-------------|
| `link` | Content for regular page links |
| `ellipsis` | Content for ellipsis indicators |

### PaginationLink

Individual page link component.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `page` | `number` | - | Page number (for static usage) |
| `isActive` | `boolean` | `false` | Whether this page is currently active |
| `class` | `string` | - | Additional CSS classes |

#### Slots

| Slot | Description |
|------|-------------|
| default | Link content (typically page number) |

### PaginationNext

Next page navigation button.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `disabled` | `boolean` | `false` | Whether the button is disabled |
| `class` | `string` | - | Additional CSS classes |

#### Slots

| Slot | Description |
|------|-------------|
| default | Button content (defaults to "Next") |

### PaginationPrevious

Previous page navigation button.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `disabled` | `boolean` | `false` | Whether the button is disabled |
| `class` | `string` | - | Additional CSS classes |

#### Slots

| Slot | Description |
|------|-------------|
| default | Button content (defaults to "Previous") |

## Alpine.js Integration

The pagination system is built around Alpine.js for reactive state management:

### Page Calculation Algorithm

The component uses an intelligent algorithm to determine visible pages:

1. **Small datasets**: Show all pages when total ≤ maxElements
2. **Start overflow**: Show first pages + ellipsis + last page 
3. **End overflow**: Show first page + ellipsis + last pages
4. **Middle overflow**: Show first + ellipsis + middle + ellipsis + last

### Event Handling

The Pagination component emits a `page-change` event when navigation occurs:

```javascript
// Listen for page changes
x-on:page-change="handlePageChange($event.detail.page)"

function handlePageChange(newPage) {
    // Update your data, make API calls, etc.
    console.log('New page:', newPage);
}
```

### External State Management

You can control pagination externally by binding to Alpine.js data:

```javascript
x-data="{ 
    currentPage: 1, 
    totalPages: 10,
    handlePageChange(page) {
        this.currentPage = page;
        // Fetch new data, update URL, etc.
    }
}"
```

## Advanced Usage

### Server-Side Integration

```javascript
// Example with URL updates
<div x-data="{
    currentPage: new URLSearchParams(window.location.search).get('page') || 1,
    handlePageChange(page) {
        this.currentPage = page;
        
        // Update URL without reload
        const url = new URL(window.location);
        url.searchParams.set('page', page);
        window.history.pushState({}, '', url);
        
        // Fetch new data
        this.fetchData(page);
    },
    fetchData(page) {
        // Your data fetching logic
    }
}">
```

## Best Practices

1. **Reasonable page sizes**: Use appropriate pageSize values (10-50 items)
2. **Clear information**: Always show current position context
3. **Consistent styling**: Maintain visual consistency with your design system
4. **Loading states**: Handle loading states during page transitions
5. **URL synchronization**: Update URLs for shareable pagination states
6. **Mobile optimization**: Consider touch-friendly button sizes
7. **Performance**: Implement efficient data fetching strategies
