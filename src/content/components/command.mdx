## Overview

The Command Palette system provides a powerful, keyboard-driven interface similar to VS Code's Command Palette (Cmd+P) or Raycast. It combines search functionality with keyboard navigation to help users quickly access actions, navigate content, and execute commands.

### Key Features

- **Fuzzy Search**: Real-time filtering of commands and content
- **Keyboard Navigation**: Full keyboard support with arrow keys and shortcuts
- **Categories**: Organize commands into logical groups
- **Recent Items**: Track and prioritize frequently used commands
- **Custom Shortcuts**: Assign keyboard shortcuts to individual commands
- **Alpine.js Integration**: Reactive state management and smooth interactions

### Component Architecture

The command palette consists of these core components:

- **Command**: Main container with dialog behavior and keyboard shortcuts
- **CommandInput**: Search input with keyboard navigation
- **CommandItem**: Individual command/action item
- **CommandItems**: Container for dynamic command lists
- **CommandFooter**: Footer with keyboard hints and actions

## Component API Reference

### Command (Main Container)

The root Command component provides the dialog container with keyboard shortcuts and state management.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `Record<string, Array<CommandItem>>` | `{}` | Categorized command items |
| `placeholder` | `string` | `'Type a command or search...'` | Input placeholder text |
| `emptyText` | `string` | `'No results found.'` | Text shown when no results match |
| `onSelect` | `string` | `'console.log(item)'` | Alpine.js expression for item selection |
| `trigger` | `'click' \| 'hover' \| 'focus' \| 'rightclick' \| 'manual'` | `'manual'` | Trigger behavior |
| `keyboardShortcut` | `string` | `'cmd+k'` | Global keyboard shortcut |
| `variant` | `'default' \| 'ghost' \| 'dialog'` | `'default'` | Visual variant |
| `size` | `'sm' \| 'md' \| 'lg' \| 'full'` | `'md'` | Size variant |
| `className` | `string` | - | Additional CSS classes |

**Command Item Structure:**

```typescript
interface CommandItem {
  title: string;        // Display name
  value: string;        // Unique identifier
  icon?: string;        // Icon name or SVG
  shortcut?: string;    // Keyboard shortcut display
  show?: boolean;       // Visibility flag
  disabled?: boolean;   // Disabled state
}
```

### CommandInput

Search input component with keyboard navigation.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `placeholder` | `string` | `'Type a command or search...'` | Input placeholder text |
| `showIcon` | `boolean` | `true` | Show search icon |
| `className` | `string` | - | Additional CSS classes |

**Keyboard Behavior:**
- Arrow Up/Down: Navigate items
- Enter: Select active item
- Escape: Clear search and close

### CommandItem

Individual command item component.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `shortcut` | `string` | - | Keyboard shortcut to display |
| `className` | `string` | - | Additional CSS classes |

**Slots:**
- `icon`: Icon content
- Default: Item content  
- `shortcut`: Custom shortcut display

### CommandItems

Container for rendering dynamic lists of command items.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'command'` | `'command'` | Styling variant |
| `className` | `string` | - | Additional CSS classes |

**Alpine.js Usage:**
Must include `x-for` and `x-key` directives for dynamic rendering.

### CommandFooter

Footer component with keyboard hints and actions.

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `showHints` | `boolean` | `true` | Show keyboard navigation hints |
| `className` | `string` | - | Additional CSS classes |

**Default Hints:**
- Enter to select
- Arrow keys to navigate  
- Escape to close

## Alpine.js State Management

The Command system integrates deeply with Alpine.js for reactive behavior:

### Global State

```javascript
// Command palette global state
window.commandState = {
    recentCommands: [],
    favoriteCommands: [],
    
    addToRecent(command) {
        this.recentCommands.unshift(command);
        this.recentCommands = this.recentCommands.slice(0, 10);
    },
    
    toggleFavorite(command) {
        const index = this.favoriteCommands.findIndex(f => f.value === command.value);
        if (index >= 0) {
            this.favoriteCommands.splice(index, 1);
        } else {
            this.favoriteCommands.push(command);
        }
    }
};
```

### Dynamic Commands

```javascript
// Dynamic command generation
x-data="{
    searchTerm: '',
    
    get dynamicCommands() {
        const commands = {};
        
        // Add recent commands if search is empty
        if (!this.searchTerm && $store.commandState.recentCommands.length) {
            commands['Recent'] = $store.commandState.recentCommands;
        }
        
        // Add filtered favorites
        if ($store.commandState.favoriteCommands.length) {
            const filtered = $store.commandState.favoriteCommands.filter(cmd =>
                cmd.title.toLowerCase().includes(this.searchTerm.toLowerCase())
            );
            if (filtered.length) commands['Favorites'] = filtered;
        }
        
        return commands;
    }
}"
```

### Command Execution

```javascript
// Command execution handler
function executeCommand(command) {
    // Add to recent commands
    window.commandState.addToRecent(command);
    
    // Execute based on command type
    switch(command.value) {
        case 'new-file':
            createNewFile();
            break;
        case 'search-files':
            openFileSearch();
            break;
        case 'git-commit':
            openGitCommitDialog();
            break;
        default:
            console.log('Unknown command:', command);
    }
}
```

## Accessibility

The Command Palette includes comprehensive accessibility features:

### Keyboard Navigation
- **Arrow Keys**: Navigate through command list
- **Enter**: Execute selected command
- **Escape**: Close palette and clear search
- **Tab**: Navigate between input and commands
- **Home/End**: Jump to first/last command

### ARIA Support
- `role="combobox"`: Identifies the search input
- `role="listbox"`: Identifies the command list
- `role="option"`: Identifies individual commands
- `aria-expanded`: Shows/hides popup state
- `aria-selected`: Indicates active command

### Screen Reader Support
- Command categories are announced
- Search results count is announced
- Selected command is announced
- Keyboard shortcuts are readable

## Best Practices

### Performance
- Use virtual scrolling for large command lists
- Implement debounced search for better performance
- Cache frequently used command results
- Lazy load command data when possible

### User Experience  
- Show recent and frequent commands first
- Provide clear visual feedback for selections
- Include helpful keyboard shortcuts
- Use consistent command naming conventions

### Development
- Keep command handlers simple and focused
- Use TypeScript for command data structure
- Implement proper error handling
- Test keyboard navigation thoroughly

The Command Palette system provides a powerful foundation for creating VS Code-style command interfaces in your Astro applications with full Alpine.js integration and accessibility support.
